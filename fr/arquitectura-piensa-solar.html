Arquitectura

Recopilación de datos

El módulo inicial se encarga de capturar toda la información que suministran los distintos dispositivos físicos del parque (inversores, strings, estaciones meteorológicas, contadores, etc.)

La comunicación e interconexión de equipos se realiza a través de un autómata (PLC)

El servidor consulta los datos actuales de los diferentes elementos utilizando el protocolo estándar de comunicación industrial ModBus TCP.

Almacenamiento de datos

Una vez capturados, los datos se insertan en una base de datos que se aloja en el servidor de la planta.

Para evitar la pérdida de información por fallos de hardware, los servidores disponen de controladoras que guardan los datos en más de un disco duro de forma simultánea.

Dispone de un módulo específico que gestiona automáticamente un backup diario de toda la información volcándola en discos duros externos.

Visualización e Interfaz de Usuario

A través de una plataforma web el usuario puede analizar tendencias, ver el estado de la planta y consultar gráficos históricos desde cualquier navegador.

La arquitectura incluye una aplicación nativa para dispositivos móviles (smartphones y tablets) con sistemas operativos iOS y Android.

La interfaz gráfica de visualización está preparada para ser multiidioma, permitiendo incorporar nuevas lenguas sin necesidad de programación adicional.

Acción e Integración Externa

Un submódulo monitoriza constantemente el parque. En caso de error o alarma, envía automáticamente alertas mediante SMS y/o correo electrónico a los usuarios según su rol.

Integra un sistema de envío de archivos vía FTP por si es necesario comunicar Piensa Solar con programas ajenos. Además, permite exportar todos los datos generados en formatos estándar como XML o CSV.
